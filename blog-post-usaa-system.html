What I Learned Building the USAA Fraud Detection System

Published February 2026

The strongest decision in this project was treating data preparation as the core deliverable, not a setup task. The model only became useful after I standardized text from several sources and removed formatting noise that broke semantic search quality.

I also learned that retrieval quality drives trust more than summary quality. Users quickly lose confidence when search returns loosely related records. Adding focus terms before embedding search improved precision and made review sessions faster for non-technical users.

For future iterations, I would measure retrieval performance more rigorously with a labeled benchmark set and tune chunking rules by document type. That should reduce false positives and make weekly risk scans more reliable.
